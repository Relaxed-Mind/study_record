-웹서버

. 웹 서버는 소프트웨어를 보통 말하지만, 웹 서버 소프트웨어가 동작하는 컴퓨터를 말한다.
. 웹 서버의 가장 중요한 기능은 클라이언트(보통 웹브라우저)가 요청하는 HTML문서나 각종 리소스를 전달하는 것
(웹서버에게 HTTP에 맞도록 리소스 요청 > 웹서버가 요청한 리소스 반환)
(만약 리소스 존재하지 않거나 전송할때 문제 발생하면 에러메세지 전송)
. 웹 브라우저나 웹 크롤러가 요청하는 리소스는 컴퓨터에 저장되어 있는 정적인 데이터이거나
동적인 결과가 될 수 있다.  
(정적: 이미지, HTML, CSS, Javascript 파일, 동적: 웹서버에 의해 실행되는 프로그램의 결과물)
웹 크롤러: 네이버나 구글같은 검색사이트에서 다른 웹사이트 정보를 읽어갈때 사용하는 소프트웨어
 
웹 서버 소프트웨어의 종류
Apache, Nginx, Microsoft, Google 웹서버
Apache: 아파치에서 개발한 웹서버로 거의 대부분의 운영체제에서 설치 및 사용 가능 (오픈소스)
Nginx: 차세대 웹서버로 더 적은 자원으로 더 빠르게 데이터를 서비스를 하는 것이 목적인 서버 (오픈소스)

-WAS(Web Application Server)

DBMS(database management system)
: 다수의 사용자들이 데이터베이스 내의 데이터를 접근할 수 있도록 해주는 소프트웨어

클라이언트 쪽에 비즈니스 로직이 많으면 클라이언트 관리, 배포에 비용 많이 발생-> 미들웨어 탄생
미들웨어
: 클라이언트와 DBMS 사이의 서버 -> 클라이언트는 입력과 출력만 담당하도록 함

WAS: 일종의 미들웨어로 웹 클라이언트의 요청 중 보통 웹 애플리케이션이 동작하도록 지원
. 기능
1. 프로그램 실행 환경과 데이터베이스 접속 기능을 제공한다.
2. 여러 개의 transaction을 관리한다. (transaction: 논리적인 작업 단위)
3. 업무를 처리하는 비즈니스 로직을 처리한다.

웹 서버 vs WAS
. WAS도 보통 자체적으로 웹 서버 기능 내장
. 규모가 커질수록 웹서버와 WAS를 분리하면 장애 극복 기능 구현 가능 (웹서버가 WAS 앞단에서 동작)
