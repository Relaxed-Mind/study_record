물리적인 하나의 컴퓨터에는 여러 개의 서버가 동작할 수 있음.
각각의 서버는 "포트"로 구분돼서 동작.
인터넷은 네트워크들의 네트워크.

HTTP(Hypertext Transfer Protocol): 서버와 클라이언트가 인터넷상에서 데이터를 주고 받기 위한 프로토콜
어떤 종류의 데이터(이미지, 동영상, 오디오, 텍스트 ....) 전송할 수 있도록 설계돼있음.

HTTP는 서버/클라이언트 모델을 따른다.
1.클라이언트가 서버에게 요청 >> 2. 서버가 클라이언트에게 응답
Stateless 방식임: 응답하고 연결 바로 끊어버림 >> 불특정 다수를 대상으로 하는 서비스에 적합
but 클라이언트의 이전상황을 알 수 없음. ->> 정보를 유지하기 위해 Cookie와 같은 기술들이 등장

URL(Uniform Resource Locator): 인터넷 상의 자원의 위치
접근 프로토콜://IP주소 또는 도메인 이름/문서의경로/문서이름
IP는 집주소, 포트는 방이라고 생각. 한 방에 한 서버.
하나의 물리적 컴퓨터에는 여러 개의 소프트웨어 서버가 동작
이 서버는 포트 값이 다르게 동작해야되고 0보다 큰 숫자값이다.
HTTP는 기본 포트값이 80번

HTTP의 요청 메세지
1. 헤더부분: 요청메소드/요청URI/HTTP버전
2. 빈줄부분
3. 바디부분

요청 메서드: GET(SELECT), PUT(UPDATE), POST(INSERT), PUSH, OPTIONS, DELETE(DELETE) . . .
요청 URI: 요청하는 자원의 위치를 명시
HTTP 버전: 웹브라우저가 사용하는 프로토콜 버전
GET은 요청바디가없음, Body는 POST나 PUT일때 들어옴.

2. 빈줄부분
3. 바디부분: 실제 응답 리소스 데이터
